# QA Handoff Checklist

Use this when handing a feature to QA for validation.

- [ ] Acceptance criteria documented and linked
- [ ] Test data and environment access provided
- [ ] Known limitations and edge cases documented
- [ ] Test plan or list of smoke tests provided
- [ ] Automation hooks or test IDs provided for regression suites
- [ ] Observability queries/dashboards for testing provided
- [ ] Security considerations or threat models shared (if applicable)

Owner: QA Lead
